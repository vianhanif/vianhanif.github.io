# Medium Prep

## Content to Copy

I've been publishing stories on my blog for a few months now — technical deep-dives, postmortems, reflections on the tools I build and the things that break. Each one goes through the same ritual: write, review, hit publish, share the link, and wait. Usually nothing comes back. No comments, no replies, no signal that anyone actually read the words.

For a long time, I wore that silence as a badge of honor. A personal blog should be for yourself, right? Not a marketing funnel. I didn't want Google Analytics-style surveillance anywhere near my site. No cookies, no trackers, no consent banners. If someone read something and got value from it, that was enough. I didn't need to know.

But the silence got heavy. Not because I wanted validation — because I wanted readers to be able to find what's worth their time. When every post in the archive looks the same (a title, a date, silence), there's no way to tell which stories connected with people. I realized that analytics and surveillance aren't the same thing. You can measure engagement without a single cookie, without knowing who anyone is.

So I'm building it: a privacy-first analytics system for my Jekyll blog. Public counts on every post — unique visitors, reading depth — not as a dashboard I hoard, but as a number on the page that says "this one resonated." The goal isn't to chase trends. It's to let people know.

This is the first post in a series where I design, build, and deploy the system. If you run a personal site and have felt the same tension between wanting signals and protecting privacy, I think you'll find something useful here.
